## Overview

This repository accompanies a paper on AI-ready transformation of Korean government budget explanation templates (`예산설명서`).

The study evaluates how **representation formats** affect the queryability and quasi-upper-bound performance of AI agents over semi-structured public budget documents. We compare five data-format-and-tool combinations on a 100-question domain-specific evaluation set drawn from 533 AI-related fiscal projects in Korea's 2026 budget.

### The 5 methods

| # | Method | Role | Input |
|---|---|---|---|
| 1 | Plain Text Vector RAG | plain-document vector baseline | natural language |
| 2 | JSON Serialization Vector RAG | serialized-JSON vector baseline | natural language |
| 3 | Text-to-Pandas Agent | structured-data agent (NL → query plan) | natural language |
| 4 | RDF Template SPARQL v3 | graph queryability quasi-upper bound | `query_logic` |
| 5 | XLSX Template Executor v4 | table queryability quasi-upper bound | `query_logic` |

### Official overall F1 (n=100, perturbed v1)

| Method | F1 (%) | F1 ≥ 0.7 |
|---|---:|---:|
| XLSX Template Executor v4 | **65.40** | 59% |
| RDF Template SPARQL v3 | **58.40** | 45% |
| Text-to-Pandas Agent | **54.57** | 47% |
| Plain Text Vector RAG | 33.77 | 23% |
| JSON Serialization Vector RAG | 14.83 | 7% |

**Interpretation**: Text-to-Pandas, RDF SPARQL, and XLSX Executor all significantly exceed the plain-text vector baseline (paired bootstrap, 95% CI). The RDF/XLSX template methods are not end-to-end natural-language QA systems; they estimate representation-level queryability under given `query_logic`.

There are two ways to run the saved-output reproduction in Colab. Path A is the fastest if you have access to the anonymous review URL.

Run the following two cells in a Colab notebook. No manual upload required.

ROOT = os.path.dirname(glob.glob("/content/**/run_reproduce_saved_outputs.sh", recursive=True)[0])

```python
# Cell 2: install dependencies and run reproduction
%cd {ROOT}
!pip install -q rdflib
!bash run_reproduce_saved_outputs.sh
```

The script verifies all required artifacts exist, reproduces the paper's headline F1 numbers (within 0.05 pp tolerance), and reproduces the paired bootstrap CIs.

### Path B — Upload ZIP via setup notebook

For Colab users who already have a local copy of the repository ZIP:

1. Open `notebooks/setup_colab.ipynb` in Colab
2. Upload the repository ZIP file (e.g., `budget-explanation-queryability.zip`) to `/content/` (Files sidebar) **or** mount Google Drive and copy the ZIP
3. Run all cells in order

The setup notebook:

* Detects and extracts the package
* Installs dependencies (`pandas`, `numpy`, `rdflib`, `openpyxl`, `matplotlib`)
* Verifies all required artifacts
* Runs the three saved-output reproduction scripts
* Cross-checks reproduced F1 values against the paper's official numbers (within 0.05 pp tolerance)

### Notes (both paths)

Total runtime: ~2 minutes. **No LLM API calls. Fully deterministic.**

The main pipeline notebook `notebooks/budget_5method_pipeline.ipynb` preserves the full experimental implementation. Full regeneration from raw data requires LLM API access and is **not** bit-level deterministic across runs (see `docs/reproducibility.md` §1).

## Quick start

> **Two reproduction paths**: this repository supports a *lightweight saved-output reproduction* (no LLM calls, requires only `pandas` and `numpy`) and a *full notebook regeneration* (requires OpenAI-compatible LLM API, embedding models, and ChromaDB). The lightweight path is recommended for review.

### Option A — Reproduce paper tables from saved outputs (no LLM calls)

The fastest verification path. Reproduces all paper score tables from saved per-question evaluation files.

```bash
pip install -r requirements.txt
bash run_reproduce_saved_outputs.sh
```

This generates `*_REPRODUCED.csv` files under `results/final/` and `results/bootstrap/`. The `validate_artifacts.py` step will fail loudly if any official F1 value diverges from the paper.

### Option B — Inspect existing results

```python
import pandas as pd

# Headline numbers (paper Table 5-1)
overall = pd.read_csv("results/final/overall_summary_5methods_perturbed_v1.csv")
print(overall)

# Statistical tests (paper Table 5-2)
boot = pd.read_csv("results/bootstrap/paired_bootstrap_5methods_perturbed_v1_percent.csv")
print(boot[["comparison", "mean_diff_pct", "ci_low_pct", "ci_high_pct", "significant_95"]])
```

### Option C — Re-run the full pipeline from raw data (Colab recommended)

Open `notebooks/budget_5method_pipeline.ipynb` and execute Cell 0 → Cell 20. Requires an OpenAI-compatible API key. Total runtime: ~30–45 min on Colab Pro. See [`docs/reproducibility.md`](docs/reproducibility.md) for the determinism caveats — commercial LLM providers are not bit-level reproducible across runs.

## Reproduction caveats

The modular Python scripts in `src/` provide deterministic saved-output validation, evaluation, and bootstrap reproduction. The LLM-dependent generation pipelines for RAG, Text-to-Pandas, JSON serialization, and RDF/SPARQL prediction generation are preserved in `notebooks/budget_5method_pipeline.ipynb` rather than refactored into standalone modules.

This repository provides a **complete deterministic saved-output reproduction path** for the reported evaluation tables and paired bootstrap results. This path does not require external LLM calls and runs in ~2 minutes.

The full generation pipeline is preserved in `notebooks/budget_5method_pipeline.ipynb`. Re-running the full pipeline may require API keys, embedding model downloads, vector index rebuilding, and saved intermediate artifacts. Because external LLM APIs and model versions may change over time, full regeneration is not guaranteed to be bit-level identical to the saved outputs.

Additional implementation caveats are documented in [`docs/reproducibility.md`](docs/reproducibility.md).
